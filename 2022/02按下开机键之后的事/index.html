按下开机键之后的事

发表于 2022-02-03 更新于 2022-02-15
# 前言 操作系统（OS）内部有许多进程管理，只要 OS 被正确加载了，那么什么软件都可以被 OS 正确地运行。 但是在计算机通电的时候，硬盘里的 OS 是怎么进入到内存并运行的呢？ 大体是这样一个流程： ![开机启动流程.jpg](http://47.96.96.172:3000/img?md5=7fc55271c9a6820690a6a110c95e0b08) # 什么叫“加载” 加载在下文的意思，基本上就是指，把储存器上的东西复制到内存里。 # 从通电到 BIOS ## BIOS 加载 是用来完成加载辅存中的任务的的程序。 它首先存在 ROM 里面。 ## ROM [ROM(Read Only Memory)](https://en.wikipedia.org/wiki/Read-only_memory) >In modern PCs, "ROM" is used to store the basic bootstrapping [firmware](https://en.wikipedia.org/wiki/Firmware "Firmware") for the processor, as well as the various [firmware](https://en.wikipedia.org/wiki/Firmware "Firmware") needed to internally control self-contained devices such as [graphic cards](https://en.wikipedia.org/wiki/Graphic_cards "Graphic cards"), [hard disk drives](https://en.wikipedia.org/wiki/Hard_disk_drive "Hard disk drive"), [solid state drives](https://en.wikipedia.org/wiki/Solid_state_drive "Solid state drive"), [optical disc drives](https://en.wikipedia.org/wiki/Optical_disc_drive "Optical disc drive"), [TFT screens](https://en.wikipedia.org/wiki/TFT_screen "TFT screen"), etc., in the system. Today, many of these "read-only" memories – especially the [BIOS](https://en.wikipedia.org/wiki/BIOS "BIOS")/[UEFI](https://en.wikipedia.org/wiki/UEFI "UEFI") – are often replaced with [EEPROM](https://en.wikipedia.org/wiki/EEPROM "EEPROM") or [Flash memory](https://en.wikipedia.org/wiki/Flash_memory "Flash memory") (see below), to permit in-place reprogramming should the need for a firmware upgrade arise. However, simple and mature sub-systems (such as the keyboard or some communication controllers in the integrated circuits on the main board, for example) may employ mask ROM or [OTP](https://en.wikipedia.org/wiki/Programmable_read-only_memory "Programmable read-only memory") (one-time programmable). > > 在现代PC中，"ROM "被用来存储处理器的基本启动固件，以及在系统中内部控制独立设备所需的各种固件，如图形卡、硬盘驱动器、固态驱动器、光盘驱动器、TFT屏幕等。今天，许多这些 "只读 "存储器--特别是BIOS/UEFI--通常被EEPROM或Flash存储器（见下文）取代，以便在需要升级固件时允许就地重新编程。然而，简单而成熟的子系统（例如键盘或主板上集成电路中的一些通信控制器）可能采用掩膜ROM或OTP（一次性可编程）。

主板上焊有一块 ROM，里面存上了 BIOS 系统，通电时，会自动映射到内存里。
现在 BIOS 已经被加载到内存里了，那么BIOS 映射在什么位置？

BIOS 位于 0xF0000 ~ 0xFFFFF (64K)。
BIOS 启动块位于 0xFE000 ~ 0xFFFFF (8K)。

CPU 的硬件设计逻辑为加电瞬间强行将 CS 的值置为 0xF000、IP 的值置为0xFFF0，即 0xFFFF0。

实模式下的内存分布

是 Intel 80286 和之后的 80x86 兼容 CPU 的操作模式（应该包括8086）。实模式有 20 根地址线，可以访问 2^20=1,048,576，即1 MB的存储器。

BIOS 到 OS 启动区

寻找启动盘

BIOS 里面有一个队列储存着接下来的启动顺序（Boot Sequence），这个排在前面的存储设备就是优先转交控制权的设备。
BIOS 会读取这个设备（也就是启动盘）的 0 盘 0 道 1 扇区（这一步在如何用-VirtualBox-运行汇编里也能看到），这个扇区有 512 个字节的内容。
如果是以 0x55、0xaa 结尾，就会被 BIOS 认作一个启动区，叫做“主引导记录”（MBR）。

问：如果第一个扇区没有符合条件的启动区，BIOS 会作何反应？是查找 0 盘 0 道 2 扇区吗？

加载 MBR

BIOS 将 MBR 加载到内存的 0x7c00 这个位置。
PC 赋值 0x7c00。
这两步的结果就是，MBR 的指令被载入了 0x7c00，CPU 下一步的就会执行 0x7c00 的指令，也就是说 CPU 的之后的运行是要受到启动盘代码的指导了。
不过现在的完整的 OS 的还没有完全被加载。

MBR 的结构：

　　（1） 第1-446字节：调用操作系统的机器码。
　　（2） 第447-510字节：分区表（Partition table）。
　　（3） 第511-512字节：主引导记录签名（0x55和0xAA）。

OS 启动区之后的事情

可以看到，从通电到加载 OS 启动区，从本质上来说，指导 CPU 将各种储存器中的内容复制到内存里，并把 PC 指向它，这里面的内容再指导下一步加载（复制）。
不同的是，ROM 和 BIOS 都是在硬件生产时就写好的。既是约定俗成，也是一种向后兼容。
OS 启动区被加载后，里面的内容会指导 OS 内核的加载，这个过程也是一种“复制”，内核运行各个程序，本质上也是一种“复制”。
